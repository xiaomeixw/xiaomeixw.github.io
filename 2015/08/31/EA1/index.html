#1.Consider providing static factory methods instead of constructors

Meaning

所谓构造器：

public class FactoryStatic {    
    public FactoryStatic() {
    }    
}

调用：

FactoryStatic instance = new FactoryStatic();

用静态工厂方法：

public class FactoryStatic {

    public static final FactoryStatic CON_INSTANCE = new FactoryStatic();

    public static FactoryStatic newInstance(){
        return FactoryStatic.CON_INSTANCE;
    }
}

调用：

FactoryStatic instance=FactoryStatic.newInstance();

Quality

advantageous

1.静态工厂方法名称可以指定具有含义的名称;

valueOf:返回的实例与它的参数具有相同的值，类型转化方法。 (或者直接用of)

getInstance:返回的实例是通过方法的参数来描述的，单例面试Singleton该方法没有参数,并返回唯一的实例;

newInstacne:同getInstance，但每次返回的实例和其他实例不同；

getType:同getInstance(),在工厂方法处于不同的类中的时候使用，type表示工厂方法所返回的对象类型；

newType:同newInstance(),在工厂方法处于不同的类中的时候使用，type表示工厂方法所返回的对象类型；

2.不必每次调用都创建新对象;

3.可以返回原返回类型的任何子类型的对象;

e.g. Java Collections framework [grepcode.com]

disadvantageous

1.类如果不含公有的或者受保护的构造器，就不能被子类化;

2.与其他的静态方法实际上没有任何区别;

#2.Builder构建器

构造器和静态工厂的问题在于无法友善的拓展大量的可选参数.
对于参数，一般采用重叠构造器的方法处理，这种过多的参数面临代码难于编写和维护.

Meaning

构造器多参：

public class People {

    private String mName;// required
    private String mSex;// required

    private String mAddress;// optional
    private String mAge;// optional
    private String mProject;// optional
    private String mFat;// optional

    public People(String name, String sex) {
        this(name, sex, "USA");
    }

    public People(String name, String sex, String address) {
        this(name, sex, address, "20");
    }

    public People(String name, String sex, String address, String age) {
        this(name, sex, address, age, "shopping,movie,travel");
    }

    public People(String name, String sex, String address, String age, 
                    String project) {
        this(name, sex, address, age, project, "0");
    }

    public People(String name, String sex, String address, String age, 
                    String project, String fat) {
        this.mName = name;
        this.mSex = sex;
        this.mAddress = address;
        this.mAge = age;
        this.mProject = project;
        this.mFat = fat;
    }

}

调用：

People people = new People("Tony", "man", " 13 Liverpool Road, 
                Islington, London, London, N1 0RW", "32", "read", "50kg");

public class Picasso {

    static volatile Picasso singleton = null;

    public static Picasso with(Context context) {
        if (singleton == null) {
          synchronized (Picasso.class) {
            if (singleton == null) {
              singleton = new Builder(context).build();              
            }
          }
        }
        return singleton;
      }

     public static class Builder {
        private final Context context;// required

        private Downloader downloader;// optional
        private ExecutorService service;// optional
        ......

        public Builder(Context context) {
          if (context == null) {
            throw new IllegalArgumentException("Context must not be null.");
          }

          this.context = context.getApplicationContext();
        }

        public Builder downloader(Downloader downloader) {
          if (downloader == null) {
            throw new IllegalArgumentException("Downloader must not be null.");
          }
          if (this.downloader != null) {
            throw new IllegalStateException("Downloader already set.");
          }
          this.downloader = downloader;
          return this;
        }

        .......

        public Picasso build() {
          Context context = this.context;

          if (downloader == null) {
            downloader = Utils.createDefaultDownloader(context);
          }

          if (service == null) {
            service = new PicassoExecutorService();
          }

#3.Singleton

Meaning

1.懒汉，线程不安全：在多线程不能正常工作

public class Singleton {  
    private static Singleton instance;  
    private Singleton (){}  

    public static Singleton getInstance() {  
    if (instance == null) {  
        instance = new Singleton();  
    }  
    return instance;  
    }  
}

2.懒汉，线程安全：能够在多线程中很好的工作，遗憾的是，效率很低，99%情况下不需要同步

public class Singleton {  
    private static Singleton instance;  
    private Singleton (){}  

    public static synchronized Singleton getInstance() {  
        if (instance == null) {  
            instance = new Singleton();  
        }  
        return instance;  
    }  
}

3.饿汉：基于classloder机制避免了多线程的同步问题，但没有达到lazy loading的目的

public class Singleton {  
    private static Singleton instance = new Singleton();  
    private Singleton (){}  
    public static Singleton getInstance() {  
        return instance;  
    }  
}

4.饿汉，变种：同2，类初始化，没有lazy loading的目的，静态域中即实例化instance

public class Singleton {
    private static  Singleton instance = null;
    static {
        instance = new Singleton();
    }

    private Singleton() {
    }

    public static Singleton getInstance() {
        return instance;
    }
}

5.静态内部类：利用了classloder的机制来保证初始化instance时只有一个线程，同时SingletonHolder类没有被主动使用，只有显示通过调用getInstance方法时，才会显示装载SingletonHolder类，从而实例化instance。起到了lazy laoding..

public class Singleton {  
    private static class SingletonHolder {  
        private static final Singleton INSTANCE = new Singleton();  
    }  
    private Singleton (){}  
    public static final Singleton getInstance() {  
        return SingletonHolder.INSTANCE;  
    }  
}

6.枚举：不仅能避免多线程同步问题，而且还能防止反序列化重新创建新的对象。

public enum Singleton {  
    INSTANCE;  
    public void whateverMethod() {  
    }  
}

7.双重校验锁：本人最爱！

public class Singleton {  
    private volatile static Singleton singleton;  
    private Singleton (){}  
    public static Singleton getSingleton() {  
        if (singleton == null) {  
            synchronized (Singleton.class) {  
            if (singleton == null) {  
                singleton = new Singleton();  
            }  
            }  
        }  
        return singleton;  
    }  
}
