【奇巧淫技004期】AndroidStudio的乱刀之策

发表于 2015-08-02

这篇奇巧淫技是关于android studio的使用，也许你很奇怪，这样的文章或者博客多了去了，为什么我还要弄成奇巧淫技，不是自己打自己脸嘛？
博主的解释：
1.网上的一坨AS的教程都是粘贴复制出来的，很多配置属性界面已经更改；我这里的AS配置都是自己在实际使用中记录的。
2.所谓的奇巧淫技不应该只是代码上的吧，我们更宽泛的去理解，工具也是奇巧一块的。

声明：本AS是最新版本，即2015年8月2日升级的1.3版本，请按照新版本的使用方式来对照。

需要了解：AS的一个work中只能放一个项目project，多个modules和库…

！！！重要的东西一定要放最前面

  1. 自动补全左边代码： alt+回车
  2. 代码提示 ： ctrl+p
  3. 自动提示 ： ctrl+alt+空格 类似eclispe中：alt+/
  4. 删除当前行：ctrl+y
  5. 负责当前行：ctrl+d
  6. 导包：ctrl+alt+o
  7. 实现方法：alt+回车
  8. get/set构造方法 alt+insert
  9. sout 控制台输出
  10. 调试不断的下一步F8

1.关闭每次打开更新检查android studio：

打开：C:\Android\Android Studio\bin\idea.properties
最后一行追加：disable.android.first.run=true

2.最简单导入eclipse项目或者库的方法：

网上有很多种导入方法,其实最简单的方法是，将eclipse的项目直接Import Module的方式导入，就可以了。这是最快速且简单的。

3.eclipse中的build automatiocally 自动编译


在android studio中是settings—compiler–make project automatically前打勾

4.设置编码


搜索关键词：File Encodings，并设置为通用的 UTF-8

5.Gradle设置


设置为AS目录下的本地gradle，同时选择offline避免反复的下载

6.模拟器的超时时间设置


搜索关键词：tasks，设置connection timeout （默认是5000ms）

7.版本控制工具


搜索关键词：version control
git要配置路径 path to git executable
svn as自己有集成

8.代码注释自动提示


搜索关键字： general
勾选show quick doc on mouse move delay 500ms

9.显示行号

Setting中：editor–general–appearance – show line numbers

10.改变java文件的文字大小

Setting中：Editor–general–colors&fonts–general 建立一个新的scheme
colors&fonts – font 修改字体大小

11.代码提示


eclipse直接alt+/就可以出来，在As中搜索关键词：code completion —》case sensitive completion 选择为none(表示代码提示不区分大小写，模糊匹配) 勾选Basic Completion(ctrl+空格)

12.自动导包


Auto import

13.设置头部说明注释

File and code Templates–》Includes–》File Header 创建属于你自己的头部注释

14.快捷键的选择：keymap 选择eclipse

建议使用default

15.设置sdk或者jdk

File—》project structure—》SDK Location

一般而言点击Title栏上的图标，可以快速进入配置

16.删除一个modules

project structure–》Modules 选择 module名然后点击删除按钮-。注意这只是表面删除。如果要磁盘都删除，可以直接进到磁盘删除，或者切换为project目录模式，点击右键delete删除

17.加载关联lib包方法

project structure —》 Dependencies —》+ 符号

  • —->library dependency 选择这里就会从jcenter中心下载lib包直接关联好；
  • —->选择 File dependency 就能够从本地的lib包里面关联
  • —->选择Mo dule dependency 直接关联同work中导入的module-lib

是否关联成功了，到你的主module中的build.gradle中去看

18.eclipse中的clean操作

在as中Build—》clean project

19.签名打包

ad中Build—》Generate signed APK

20.新建lib库文件

在elipse中是直接勾选，而在As中是在new module –》android library然后再下一步

或者直接在该module的修改build.gradle的apply plugin为：

apply plugin: 'com.android.library'
