#解析MVP

Model层：java-bean对象，实现序列化Parcelable

Presenter层：基类Presenter

public interface Presenter<V> {
    void attachView(V view);
    void detachView();
}

实现者：

public class MainPresenter implements Presenter<MvpView> {
    //View对象
    private MvpView view;

    //绑定view
    @Override
    public void attachView(MvpView view) {
        this.view = view;
    }

    //解绑View
    @Override
    public void detachView() {
        view=null;
    }

    //该方法到Activity中调用
    public void loadRepositories() {
        //请求数据
        rx.send(
            onSucced(){
                //把结果传递给view并通知view更新
                view.showRepositories(list);
            }

            onFail(){
                view.showMessage(string);
            }

        );

    }    

}

View层：基类View

public interface MvpView {
    Context getContext();
}

继承者：展示要变化界面的view动态方法

public interface MainMvpView extends MvpView {
    void showRepositories(List<?> list);
    void showMessage(string error);
}

Activity中实现其对应的View的方法

public class MainActivity extends Activity implements MainMvpView {

    private MainPresenter presenter;

    @Override
    protected void onCreate(Bundle savedInstanceState) {
        super.onCreate(savedInstanceState);
        //1.view指向绑定presenter
        presenter = new MainPresenter();
        presenter.attachView(this);
        //2.加载xml布局
        setContentView(R.layout.activity_main);
        infoTextView = (TextView) findViewById(R.id.text_info);

        //3.findViewById
        reposRecycleView = (RecyclerView) findViewById(R.id.repos_recycler_view);
        setupRecyclerView(reposRecycleView);

        //4.拿数据并更新界面
        setupPresenter(){
            //该方法在presenter中完成后台请求，并在请求完成后，调用到View中的loadRepositories，而loadRepositories方法的具体实现则是此activity 实现View的具体方法，也就是下面的那个@Override showRepositories方法
            presenter.loadRepositories();
        };
    }


    @Override
    public Context getContext() {
        return this;
    }

    @Override
    public void showRepositories(List<?> list) {
        //拿到请求的数据界面来更新

    }

     @Override

}

1.Activity实现于View;

2.Activity调用presenter请求数据;

3.presenter中请求数据,并将json–解析–》model-bean对象;

4.presenter中请求成功(失败/加载中)后,调用view.onsucced(fail/loading)方法通知UI更新;

5.activity的onsucced(fail/loading)方法完成UI更新;
